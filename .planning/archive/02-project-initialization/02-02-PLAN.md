```

   - Check if `.planning/PROJECT.md` already exists
   - If yes: warn the user that re-initializing will overwrite existing project files
   - Use AskUserQuestion: "A project already exists. Reinitialize?" with options:
     - "Yes, start fresh" — proceed, will overwrite
     - "No, keep existing" — abort with message to use /agency:status instead
   - If no existing project: proceed directly

2. ENSURE DIRECTORY STRUCTURE
   - Create `.planning/` directory if it doesn't exist
   - Create `.planning/phases/` directory if it doesn't exist

3. RUN QUESTIONING FLOW (Stage 1: Vision & Identity)
   - Follow the questioning-flow skill's Stage 1
   - Ask "What are you building? Give me the elevator pitch."
   - Follow up based on response to capture: project name, description, value prop, target users
   - Summarize and confirm understanding before moving on

4. RUN QUESTIONING FLOW (Stage 2: Requirements & Constraints)
   - Follow the questioning-flow skill's Stage 2
   - Capture: must-have features, out of scope, constraints
   - Adapt questions based on project type (code vs content vs design vs mixed)
   - Note any key decisions made during discussion

5. RUN QUESTIONING FLOW (Stage 3: Workflow Preferences)
   - Use AskUserQuestion for structured choices (from questioning-flow skill):
     - Execution mode: Guided (default) / Autonomous / Collaborative
     - Planning depth: Standard (default) / Quick Sketch / Deep Analysis
     - Cost profile: Balanced (default) / Economy / Premium

6. GENERATE PROJECT.MD
   - Read `.planning/templates/project-template.md`
   - Fill all placeholders with questioning output (per the output mapping in questioning-flow skill)
   - Write to `.planning/PROJECT.md`
   - Omit sections that don't apply (don't write "N/A")

7. DECOMPOSE INTO PHASES AND GENERATE ROADMAP.MD
   - Analyze requirements from Step 4
   - Follow phase decomposition guidelines from questioning-flow skill
   - For each phase:
     - Define goal and requirements covered
     - Use agent-registry recommendation algorithm to suggest agents
     - Write testable success criteria
     - Estimate plan count (2-3 per phase)
   - Read `.planning/templates/roadmap-template.md`
   - Fill placeholders with phase breakdown
   - Write to `.planning/ROADMAP.md`

8. GENERATE STATE.MD
   - Read `.planning/templates/state-template.md`
   - Fill placeholders: total phases, initial position (phase 0), date, first phase name
   - Include workflow preferences in "Recent Decisions" section
   - Write to `.planning/STATE.md`

9. DISPLAY SUMMARY
   - Show the user a summary of what was created:
     - Project name and one-line description
     - Number of phases with names
     - Workflow preferences chosen
     - Next action: "Run `/agency:plan 1` to begin Phase 1: {first_phase_name}"
   - Do NOT dump the full file contents — just the summary
```
