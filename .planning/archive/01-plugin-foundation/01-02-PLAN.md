**Section 3: Recommendation Algorithm**
Document the agent recommendation process (to be used by /agency:plan and /agency:quick):

1. **Parse task description** — Extract key terms and intent
2. **Match task types** — Find agents whose task_types overlap with extracted terms
3. **Rank by relevance** — Primary division match > secondary cross-division match
4. **Cap recommendations** — Recommend 2-4 agents per task (avoid agent sprawl)
5. **Include rationale** — For each recommended agent, explain WHY they fit
6. **Always include one tester** — Every execution team should have at least one testing agent for QA
7. **Cross-division triggers** — If task spans multiple divisions, recommend a project-manager or agents-orchestrator as coordinator
