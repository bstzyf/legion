Create the `.claude-plugin/` directory and `plugin.json` manifest.

The manifest must contain these fields per PLUG-01:

```json
{
  "name": "agency-workflows",
  "version": "2.0.0",
  "description": "Orchestrate 51 AI specialist personalities as coordinated teams — project initialization, phase planning, parallel execution, quality review, and portfolio management",
```

Field rationale:
- **name**: `agency-workflows` — kebab-case, becomes the skill namespace prefix (`/agency-workflows:skill-name`). Descriptive of what the plugin does.
- **version**: `2.0.0` — this is the v2.0 milestone introducing plugin format.
- **description**: One sentence covering the core value proposition and key capabilities.
- **author**: GitHub username and profile URL. Email omitted (optional field).
- **repository**: GitHub repo URL for the project.
- **license**: MIT (standard open-source license).
- **keywords**: Discovery-relevant terms for marketplace search.

IMPORTANT: The `.claude-plugin/` directory should contain ONLY `plugin.json`. No other files go here — the plugin docs are explicit about this.

<task type="auto">
  <name>Task 2: Create plugin settings</name>
  <files>settings.json</files>
  <action>
Create `settings.json` at the plugin root (NOT inside `.claude-plugin/`).

Content:
```json
{}
```

Why an empty object:
- Claude Code plugin `settings.json` currently only supports the `agent` key, which activates a single agent as the main thread
- Agency is a multi-agent orchestration system accessed through 9 commands and 15 skills — it does not operate as a single-agent tool
- Setting an `agent` key would force one of the 51 agents as the default, which contradicts the dynamic agent selection design
- The empty object satisfies PLUG-05 (settings file exists with valid default configuration) while preserving the multi-agent design
- Unknown keys are silently ignored, so the file is future-compatible — when Claude Code adds more settings keys, Agency can adopt them

The file MUST be valid JSON (not empty, not a comment — a literal `{}`).
  </action>
  <verify>
Verify settings.json is valid JSON:
```bash
cat settings.json | python3 -c "import sys,json; json.load(sys.stdin); print('Valid JSON')"
```
  </verify>
  <done>settings.json exists at plugin root with empty object `{}`. Valid JSON, satisfies PLUG-05.</done>
</task>

Create three empty directories at the plugin root with `.gitkeep` files so Git tracks them:

```bash
mkdir -p commands skills agents
touch commands/.gitkeep skills/.gitkeep agents/.gitkeep
```

These directories are the migration targets:
- **`commands/`** — Phase 18 migrates 9 command `.md` files here from `.claude/commands/agency/`
- **`skills/`** — Phase 17 migrates 15 skills here as `{name}/SKILL.md` subdirectories from `.claude/skills/agency/`
- **`agents/`** — Phase 16 migrates 51 agent `.md` files here from `agency-agents/{division}/`

The `.gitkeep` files ensure Git tracks the empty directories. They will be automatically superseded when actual files are migrated in.

IMPORTANT: These directories go at the PROJECT ROOT, alongside `.claude-plugin/` and `settings.json`. They are NOT inside `.claude-plugin/`.
