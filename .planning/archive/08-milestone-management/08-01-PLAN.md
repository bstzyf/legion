**Section 1: Milestone Format**

Define the milestone schema as it appears in ROADMAP.md. Milestones are added as a `## Milestones` section AFTER the `## Phase Details` section and BEFORE the `## Progress` table.

```markdown
## Milestones

### Milestone {N}: {name}
- **Phases**: {start}-{end}
- **Goal**: {one-line milestone goal}
- **Status**: Pending | In Progress | Complete | Archived
- **Completed**: {YYYY-MM-DD} (only present when Status is Complete or Archived)
```

Rules:
- Milestone numbers are sequential starting from 1
- Phase ranges are inclusive and must not overlap (e.g., Phases 1-7 and Phases 8-14, not 1-7 and 7-14)
- Phase ranges must cover ALL phases in the roadmap — no gaps allowed
- Every phase belongs to exactly one milestone
- Status meanings:
  - **Pending**: No phases in this milestone have started execution
  - **In Progress**: At least one phase has started but not all are Complete
  - **Complete**: All phases in the milestone are marked Complete in the Progress table
  - **Archived**: All phases archived to `.planning/archive/milestone-{N}/`
- The **Completed** field is added when status transitions to Complete, and preserved when transitioning to Archived

**Section 2: Milestone Definition**

How to propose and define milestones when they don't exist in ROADMAP.md yet.

**Auto-propose milestones**:
1. Read ROADMAP.md phase list
2. Analyze phases for logical groupings based on:
   - Shared theme (e.g., "core workflow", "advanced features", "integrations")
   - Dependency clusters (phases that build on each other)
   - Natural break points (major deliverables or user-facing milestones)
3. Propose 2-4 milestones (aim for 3-7 phases per milestone)
4. Each proposal includes: name, phase range, one-line goal
5. Present proposals to user via AskUserQuestion for confirmation
6. User can accept, modify boundaries, rename, or define from scratch

**Write milestones to ROADMAP.md**:
1. Read current ROADMAP.md
2. Insert `## Milestones` section after `## Phase Details` and before `## Progress`
3. Add each milestone as `### Milestone {N}: {name}` with fields
4. Derive initial status for each milestone:
   - Check each phase in the range against the Progress table
   - If all Complete → status = Complete
   - If any started → status = In Progress
   - If none started → status = Pending
5. Write updated ROADMAP.md

**Section 3: Milestone Completion**

How to check and mark a milestone as complete.

**Pre-flight checks**:
1. Read ROADMAP.md — verify `## Milestones` section exists
2. Identify the target milestone (either specified by number or auto-detected as the earliest In Progress milestone)
3. Read the Progress table — check every phase in the milestone's range:
   - All phases must show Status = "Complete" in the Progress table
   - If any phase is not Complete: report which phases are incomplete, do NOT mark milestone complete
4. Verify all phases passed review (check for UAT.md or review status in STATE.md)

**Generate milestone summary**:
1. Create directory `.planning/milestones/` if it doesn't exist
2. Gather metrics from across the milestone's phases:
   - **Total plans**: sum of Plans column for phases in range
   - **Plans completed**: sum of Completed column for phases in range
   - **Requirements satisfied**: count requirements marked `[x]` that map to phases in range (from REQUIREMENTS.md traceability table)
   - **Phases**: count of phases in the milestone
   - **Key files created/modified**: scan SUMMARY.md files for each phase, extract file lists
   - **Agents used**: scan CONTEXT.md files for agent recommendations, deduplicate
3. Gather qualitative data:
   - **Key outcomes**: extract the `<done>` text from each plan's SUMMARY.md (or the summary bullet points)
   - **Decisions made**: collect from STATE.md "Recent Decisions" that relate to phases in the milestone range
4. Write `.planning/milestones/MILESTONE-{N}.md`:

```markdown
# Milestone {N}: {name} — Summary

## Completed
{YYYY-MM-DD}

## Goal
{milestone goal from ROADMAP.md}

## Metrics
| Metric | Value |
|--------|-------|
| Phases | {count} |
| Plans completed | {completed}/{total} |
| Requirements satisfied | {count} |
| Key files | {count} created/modified |

## Phase Outcomes
| Phase | Name | Plans | Key Deliverable |
|-------|------|-------|-----------------|
| {N} | {name} | {count} | {one-line from SUMMARY} |
...

## Key Decisions
- {decision 1}
- {decision 2}
...

## Files Created
- {list of key files created across all phases, grouped by type}

## Archive Location
.planning/archive/milestone-{N}/
```

**Mark milestone complete**:
1. Read ROADMAP.md
2. Update the milestone's Status from "In Progress" to "Complete"
3. Add `- **Completed**: {YYYY-MM-DD}` field
4. Write updated ROADMAP.md

**Update STATE.md**:
1. Read STATE.md
2. Add or update a `## Milestones` section (after Recent Decisions):
   - `- Milestone {N}: {name} — Complete ({date})`
3. Write updated STATE.md

**Section 4: Milestone Archiving**

How to archive a completed milestone's phase artifacts.

**Pre-flight checks**:
1. Milestone must have Status = "Complete" (not Pending, In Progress, or already Archived)
2. Milestone summary must exist at `.planning/milestones/MILESTONE-{N}.md`
3. All phase directories in the milestone's range must exist in `.planning/phases/`

**Archive operations**:
1. Create `.planning/archive/milestone-{N}/` directory
2. For each phase in the milestone's range:
   a. Source: `.planning/phases/{NN-name}/`
   b. Destination: `.planning/archive/milestone-{N}/{NN-name}/`
   c. Move the entire directory (preserving all files: CONTEXT.md, PLAN.md, SUMMARY.md, UAT.md)
3. Verify the move: check that destination directories exist and source directories are gone

**Update state files after archiving**:

ROADMAP.md:
- Update the milestone's Status from "Complete" to "Archived"
- In the Progress table, append " (Archived)" to the Status of each archived phase

STATE.md:
- Condense Phase Results sections for archived phases:
  Replace each `## Phase {N} Results` block with:
  `## Phase {N} Results — Archived`
  `See .planning/milestones/MILESTONE-{M}.md and .planning/archive/milestone-{M}/{NN-name}/`
- Update the Milestones section: `- Milestone {N}: {name} — Archived ({date})`

Milestone summary:
- Update `.planning/milestones/MILESTONE-{N}.md`:
  Add to Archive Location section: `Archived on {YYYY-MM-DD}`

**Section 5: Milestone Metrics**

Formulas for computing milestone-level metrics. These are used by both the completion summary and the `/agency:milestone` status display.

```
Milestone progress:
  phases_complete = count of phases in range with Status = "Complete" in ROADMAP.md Progress table
  phases_total = end - start + 1 (inclusive range)
  percentage = floor((phases_complete / phases_total) * 100)

Milestone plan progress:
  plans_complete = sum of Completed column for phases in range
  plans_total = sum of Plans column for phases in range
  plan_percentage = floor((plans_complete / plans_total) * 100)

Overall milestone progress (for display):
  Use plan_percentage as the primary metric (more granular than phase count)
  Progress bar: 10 characters wide
  filled = floor(plan_percentage / 10)
  empty = 10 - filled
  Format: [{filled}{empty}] {plan_percentage}%
```

Requirement coverage:
1. Read REQUIREMENTS.md traceability table
2. Find all requirement IDs mapped to phases in the milestone's range
3. Count how many are checked `[x]` vs total
4. Format: `{checked}/{total} requirements satisfied`

**Section 6: Error Handling**

- **No ## Milestones section in ROADMAP.md**: Offer to define milestones (Section 2). Do not error — this is an expected state for projects that haven't set up milestones yet.
- **Incomplete phases in milestone**: Report exactly which phases are incomplete. List their current status. Do NOT mark milestone complete.
- **Missing phase directories during archive**: Report the missing directories. Archive what exists, note gaps in the milestone summary.
- **Already archived milestone**: Report "Milestone {N} is already archived at .planning/archive/milestone-{N}/". Do not re-archive.
- **Milestone summary already exists**: During completion, check if `.planning/milestones/MILESTONE-{N}.md` exists. If so, ask user whether to overwrite or skip summary generation.
- **Archive directory already exists**: If `.planning/archive/milestone-{N}/` exists (partial previous archive?), ask user whether to continue (move remaining dirs) or abort.
- **Permission errors on directory moves**: Report clearly with the exact source and destination paths that failed.
- **Phase range gaps or overlaps**: During definition, validate ranges cover all phases exactly. Report gaps or overlaps and ask user to fix.
