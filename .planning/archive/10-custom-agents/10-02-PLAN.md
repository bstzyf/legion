Create `.claude/commands/agency/agent.md` -- the command entry point for `/agency:agent`. Follow the established command pattern from start.md and milestone.md. Target 80-120 lines.

**Frontmatter:**
```yaml
---
name: agency:agent
description: Create a new agent personality through a guided workflow
allowed-tools: [Read, Write, Edit, Bash, Grep, Glob, AskUserQuestion]
---
```

**Objective block:**
```
Guide the user through an adaptive conversation to define a new agent personality.
Validate the schema, generate the agent .md file, and register the new agent
in the agent-registry so it appears in future planning recommendations.

Purpose: Single command for creating custom agents with validation and registry integration.
Output: New agent .md file in agency-agents/{division}/, updated agent-registry.md.
```

**Execution context:**
```
@./.claude/skills/agency/workflow-common.md
@./.claude/skills/agency/agent-registry.md
@./.claude/skills/agency/agent-creator.md
```

**Context:**
Reference a few existing agent files as structural examples:
```
@agency-agents/engineering/engineering-senior-developer.md
@agency-agents/testing/testing-reality-checker.md
@agency-agents/design/design-ui-designer.md
```

**Process (8 steps):**

1. PRE-FLIGHT CHECK
   - Check if `.planning/PROJECT.md` exists by attempting to read it
   - If not found: display "No Agency project found. Run `/agency:start` to initialize." and exit
   - Read `.planning/STATE.md` for current project context

2. LOAD REGISTRY CONTEXT
   - Read agent-registry.md to understand existing agents, divisions, and task type taxonomy
   - Read workflow-common.md for personality paths and division list
   - List existing agents: `ls agency-agents/` to confirm division directories

3. STAGE 1: AGENT IDENTITY
   - Follow agent-creator.md Section 2, Stage 1 exactly
   - Open with the identity question
   - Extract specialty, infer division and name
   - Confirm via AskUserQuestion
   - If user adjusts, update inferred values

4. STAGE 2: CAPABILITIES AND PERSONALITY
   - Follow agent-creator.md Section 2, Stage 2 exactly
   - Ask capability, personality, and hard-rules questions
   - Capture responses, summarize, confirm via AskUserQuestion

5. STAGE 3: REGISTRY TAGS
   - Follow agent-creator.md Section 2, Stage 3 exactly
   - Present suggested tags based on capabilities
   - Show existing tags from the registry for alignment
   - Confirm via AskUserQuestion

6. VALIDATE SCHEMA
   - Follow agent-creator.md Section 3 exactly
   - Run all validation checks (name uniqueness, format, description, color, division, body minimum)
   - If any check fails: present errors, allow correction, re-validate
   - Do NOT proceed until all validations pass

7. GENERATE FILES
   - Follow agent-creator.md Section 4 (file generation) then Section 5 (registry update)
   - Write the agent .md file to `agency-agents/{division}/{agent-name}.md`
   - Verify file exists after write
   - Update agent-registry.md with the new catalog row
   - If either fails: follow Section 6 error handling

8. CONFIRM AND COMMIT
   - Display the summary from agent-creator.md Section 7
   - Create a git commit with message: `feat(agents): add custom agent {agent-name}`
   - Files to commit: `agency-agents/{division}/{agent-name}.md` and `.claude/skills/agency/agent-registry.md`
   - Update STATE.md: add a line under Recent Decisions noting the custom agent creation

**Change 1: Section 1 intro update**

```
Complete catalog of all 51 agent personalities across 9 divisions.
```

Replace with:
```
Complete catalog of all agent personalities. Includes 51 built-in agents across 9 divisions plus any custom agents created via `/agency:agent`.
```

**Change 2: Add custom agent guidance at end of Section 1**

After the last division table in Section 1 (after the Specialized Division table and before the `---` separator before Section 2), add:

```markdown
### Custom Division

Custom agents created via `/agency:agent` appear here. This section is populated automatically by the agent-creator skill.

| ID | File | Specialty | Task Types |
|----|------|-----------|------------|

*No custom agents yet. Run `/agency:agent` to create one.*
```

This gives the agent-creator skill (Plan 01's Section 5) a clear insertion point -- it appends rows to this table and removes the "No custom agents yet" note when the first agent is added.

**Change 3: Add note to Section 3 Step 1**

In Section 3, Step 1 ("Parse Task Description"), after the existing text, add a note:

```
**Custom agents:** Custom agents added to the Section 1 catalog via `/agency:agent` are automatically eligible for recommendation. Their task type tags are matched identically to built-in agents. No changes to the scoring algorithm are needed.
```

This makes explicit what the research document stated: custom agents participate in the recommendation algorithm by virtue of being in the Section 1 catalog.

Do NOT change the recommendation algorithm itself -- the research confirms that adding rows to Section 1 is sufficient for custom agents to be scored and recommended.
