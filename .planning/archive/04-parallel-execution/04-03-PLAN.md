**process** — Fully specified steps (not placeholders):

```
1. DETERMINE TARGET PHASE
   - Check $ARGUMENTS for --phase N flag
   - If no flag: read STATE.md to determine current phase
     - Use the phase number from "Phase: N of M" in Current Position
     - If status says "planned" or "pending": that's the target
     - If status says "executed" or "complete": error — "Phase {N} already executed. Run /agency:plan {N+1} for next phase."
   - Validate: phase must exist in ROADMAP.md and have plan files
   - If no plans exist: error — "No plans found for Phase {N}. Run /agency:plan {N} first."

2. DISCOVER PLANS
   Follow wave-executor skill Section 2 (Plan Discovery):
   - Read the phase directory: .planning/phases/{NN}-{slug}/
   - Find all {NN}-{PP}-PLAN.md files
   - Parse YAML frontmatter from each plan
   - Build the wave map: group plans by wave number
   - Validate: no circular dependencies, all depends_on references valid
   - Display plan summary to user:
     "Phase {N}: {name} — {plan_count} plans across {wave_count} waves"
     For each wave: list plan numbers and names

3. PRE-EXECUTION CONFIRMATION
   - Show the user what's about to execute:
     - Phase name and number
     - Number of plans and waves
     - Agents that will be spawned (from plan frontmatter)
     - Files that will be modified (aggregated from all plans)
   - Use AskUserQuestion: "Ready to execute Phase {N}?"
     - "Execute all plans" — proceed
     - "Execute specific wave only" — ask which wave, execute just that wave
     - "Cancel" — abort, no changes made

4. EXECUTE WAVES
   Follow wave-executor skill Section 4 (Wave Execution):
   For each wave (sequential):
   a. Announce: "Starting Wave {W} — {plan_count} plan(s)"
   b. For each plan in the wave:
      - Follow wave-executor Section 3 (Personality Injection)
      - Read agent personality .md file (if not autonomous)
      - Construct the agent prompt with personality + plan
      - Spawn agent via Agent tool with model: "sonnet"
   c. Spawn all wave agents in parallel (multiple Agent calls in one message)
   d. Wait for all agents to complete
   e. Process results: follow wave-executor Section 5 (Agent Result Processing)
      - Write summary files for each completed plan
   f. Track progress: follow execution-tracker Section 2 (Plan Completion)
      - Update STATE.md for each plan
      - Create atomic git commit for each successful plan
   g. Track wave: follow execution-tracker Section 3 (Wave Completion)
      - Update ROADMAP.md progress table
      - Report wave status
   h. If ANY plan in the wave failed: STOP
      - Do NOT proceed to next wave
      - Report failures with summary references
      - Suggest: "Run /agency:review to diagnose failures"

5. COMPLETE PHASE EXECUTION
   Follow execution-tracker Section 4 (Phase Completion):
   - Calculate final phase status (all passed vs. some failed)
   - Update STATE.md with phase completion status
   - Update ROADMAP.md progress table
   - Display final progress with plan-by-plan results table

6. ROUTE TO NEXT ACTION
   - If all plans passed:
     "Phase {N}: {name} complete. Run `/agency:review` to verify the work."
   - If some plans failed:
     "Phase {N}: {name} partial — {count} plan(s) failed.
      Review summaries: .planning/phases/{NN}-{slug}/{NN}-{PP}-SUMMARY.md
      Then run `/agency:review` for diagnosis."
   - Do NOT automatically trigger /agency:review — let the user decide when
```

IMPORTANT:
- The process steps must be detailed enough that Claude can execute them without ambiguity
- Each step references the specific skill section it follows
- The command handles both the happy path (all plans pass) and failure path (some plans fail)
- Pre-execution confirmation prevents accidental execution
- Wave-by-wave progress is visible to the user throughout
- AskUserQuestion is used for the execution confirmation gate
