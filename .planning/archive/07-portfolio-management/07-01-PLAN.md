**Section 1: Portfolio Registry Format**

Define the PORTFOLIO.md file format stored at `~/.claude/agency/portfolio.md`:

```markdown
# Agency Portfolio

## Projects

### {project-name}
- **Path**: {absolute-path-to-project-root}
- **Status**: {Active | Paused | Complete | Stale}
- **Registered**: {YYYY-MM-DD}
- **Description**: {one-line description from PROJECT.md}

### {another-project}
...

## Cross-Project Dependencies

| ID | From | To | Type | Status | Notes |
|----|------|----|------|--------|-------|
| DEP-01 | {project-a}:Phase {N} | {project-b}:Phase {M} | blocks | Active | {why} |

## Metadata
- **Last Updated**: {YYYY-MM-DD}
- **Total Projects**: {count}
- **Active Projects**: {count}
```

Rules:
- Each project has a unique heading (### {project-name}) — the name from its PROJECT.md
- If two projects have the same name, append directory name in parentheses
- Status meanings:
  - **Active**: Project directory exists and has valid .planning/ state
  - **Paused**: User manually marked as paused
  - **Complete**: All phases complete in ROADMAP.md
  - **Stale**: Project directory not found at registered path
- Dependencies use `{project-name}:Phase {N}` notation
- Dependency types: `blocks` (hard), `informs` (soft — output useful but not required)

**Section 2: Registry Operations**

Define these operations (instructions for commands to follow, not function signatures):

**Register Project**:
1. Read `~/.claude/agency/portfolio.md` (create if doesn't exist, including `~/.claude/agency/` directory)
2. Read the current project's `.planning/PROJECT.md` — extract project name and description
3. Get the absolute path of the current working directory
4. Check if this project is already registered (match by path):
   - If already registered: update the name/description if changed, log "Project already in portfolio"
   - If not registered: add a new `### {project-name}` entry
5. Update the Metadata section (Last Updated, Total Projects, Active Projects counts)
6. Write the updated PORTFOLIO.md

**Unregister Project**:
1. Read `~/.claude/agency/portfolio.md`
2. Find the project entry matching the current working directory path (or a specified project name)
3. Remove the `### {project-name}` section
4. Remove any dependencies referencing this project
5. Update Metadata section
6. Write the updated PORTFOLIO.md

**List Projects** (for dashboard assembly):
1. Read `~/.claude/agency/portfolio.md`
2. Parse each `### {project-name}` section
3. For each registered project:
   a. Check if the path exists (filesystem check)
   b. If path exists: read `{path}/.planning/STATE.md` and `{path}/.planning/ROADMAP.md`
   c. Extract: current phase, phase count, completion %, status, last activity
   d. If path doesn't exist: mark as Stale
4. Return the assembled project list with live state

**Section 3: State Aggregation**

When reading cross-project state for the dashboard:

**Per-project state extraction** (from STATE.md):
- Current phase number and total phases
- Status text
- Last activity date and description
- Next action text

**Per-project progress extraction** (from ROADMAP.md):
- Phase list with completion status
- Plans completed vs total
- Overall progress percentage (use execution-tracker Section 5 formula)

**Health assessment rules**:
- Green: Phase on track, no blockers in STATE.md, progress advancing
- Yellow: STATE.md mentions "escalated", "failed", or "blocked"; or no activity in 7+ days
- Red: Project directory missing (Stale), or STATE.md has unresolved blockers for 3+ phases

**Aggregated portfolio metrics**:
- Total projects (Active count)
- Overall progress (average completion % across active projects)
- Projects at risk (Yellow + Red count)
- Cross-project dependency health (all resolved vs some blocking)

**Section 4: Cross-Project Dependencies**

Operations for managing dependencies between projects:

**Add Dependency**:
1. Read PORTFOLIO.md
2. Validate both projects exist in registry
3. Assign next DEP-{NN} ID
4. Add row to Cross-Project Dependencies table
5. Write PORTFOLIO.md

**Remove Dependency**:
1. Read PORTFOLIO.md
2. Find dependency by ID (or by from/to match)
3. Remove row from table
4. Write PORTFOLIO.md

**Check Dependencies**:
1. Read PORTFOLIO.md dependencies table
2. For each active dependency:
   a. Read the "From" project's ROADMAP.md
   b. Check if the referenced phase is complete
   c. If complete: dependency is resolved
   d. If not complete: dependency is blocking
3. Return list of blocking dependencies with impact assessment

Dependency format: `{project-name}:Phase {N}` — references a specific phase in a specific project.

**Section 5: Agent Allocation**

Derive agent usage across projects at read-time:

**Scan agent assignments**:
1. For each registered active project:
   a. Read all `{path}/.planning/phases/*/` directories
   b. **Primary source**: Read CONTEXT.md files — these contain explicit agent recommendations per phase
   c. **Secondary source**: Scan plan body text and SUMMARY.md files for agent ID references (e.g., "engineering-senior-developer", "testing-reality-checker")
   d. Note: YAML frontmatter in plan files does NOT contain agent IDs — do not rely on frontmatter for allocation data
2. Build a map: `{agent-id} -> [{project-name}, ...]`

**Allocation summary**:
- Most-used agents (agents assigned to 2+ projects)
- Division spread (which divisions are active across the portfolio)
- Underutilized divisions (divisions with 0 projects using their agents)

Note: Since agents are personality files (not running processes), "allocation" is about visibility into usage patterns, not resource locking. Any project can use any agent at any time.

**Section 6: Error Handling**

- **Missing PORTFOLIO.md**: Create with empty structure on first register
- **Missing project directory**: Mark project as Stale, don't remove (user may want to update path)
- **Corrupted STATE.md/ROADMAP.md in a project**: Skip that project in aggregation, report as "Unable to read state"
- **Permission errors on ~/.claude/**: Report clearly — "Cannot create portfolio registry at ~/.claude/agency/. Check permissions."
- **Name collisions**: Append directory basename in parentheses to disambiguate
- **Empty portfolio**: Dashboard shows "No projects registered. Run /agency:start in a project to add it."

```markdown
The portfolio registry lives at `~/.claude/agency/portfolio.md` — outside any project directory. This file is shared across all Agency projects on the machine.

Projects are auto-registered in the portfolio when `/agency:start` completes. Registration stores the project name, absolute path, registration date, and one-line description.

Portfolio commands read each registered project's `.planning/STATE.md` and `.planning/ROADMAP.md` at request time. There is no background sync. If a project directory is missing, it's marked Stale.

### Portfolio Command Convention
| Command | Purpose | Cost Tier |
|---------|---------|-----------|
| `/agency:portfolio` | Multi-project dashboard and dependency management | Haiku (dashboard), Opus (Studio Producer insights) |
```
