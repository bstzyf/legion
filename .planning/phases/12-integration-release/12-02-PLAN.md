Bump the version to 6.0.0, regenerate checksums for all publishable files, and run the full release check to verify all gates pass. This is the final step before v6.0.0 can be published.
</objective>

<tasks>
## Task 1: Version Bump to 6.0.0

**Action**: Update version across all version-bearing files.

1. Read `package.json` — change `"version": "5.0.0"` to `"version": "6.0.0"`
2. Update description in `package.json` if it doesn't reflect v6.0 capabilities (optional — only if meaningfully outdated)
3. Read `.claude-plugin/plugin.json` — update version to `6.0.0`
4. Search for any other files referencing version `5.0.0` that need updating
5. Do NOT modify STATE.md version references — those are historical records

**Verify**: `node -e "const p = require('./package.json'); console.log(p.version)"` outputs `6.0.0`

**Done when**: All version-bearing files show 6.0.0.

## Task 2: Regenerate Checksums

**Action**: Regenerate checksums.sha256 with current file hashes.

1. Run `node scripts/generate-checksums.js`
2. Verify the output file `checksums.sha256` was updated
3. Spot-check: verify a few key files appear in the checksums (agents/, commands/, skills/)
4. Ensure no stale entries remain for deleted files

**Verify**: `checksums.sha256` exists and contains entries for all publishable files

**Done when**: checksums.sha256 is freshly generated with correct hashes for all current files.

## Task 3: Release Check Validation

**Action**: Run the full release check and fix any failures.

1. Run `node scripts/release-check.js`
2. If any checks fail:
   a. Read the error messages carefully
   b. Fix the root cause (not the symptom)
   c. Re-run until all checks pass
3. Run the full prepublishOnly script: `bash scripts/validate.sh && node scripts/release-check.js && node --test`
4. Record the final clean output

**Verify**: `node scripts/release-check.js` exits with code 0; all gates pass

**Done when**: Release check passes all gates, prepublishOnly succeeds, v6.0.0 is ready to publish.
</tasks>
