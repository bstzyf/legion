The section must include:

**7.1 parseNaturalLanguage(input)**
A function that takes a raw text string and returns a structured result:
```javascript
/**
 * @param {string} input - Raw user text (e.g., "fix the tests", "harden security")
 * @returns {Object} { command, flags, confidence, fallbackSuggestion }
 *
 * confidence levels:
 * - HIGH (>= 0.8): auto-route to command
 * - MEDIUM (0.5-0.79): suggest command, ask user to confirm
 * - LOW (< 0.5): show top 3 suggestions, ask user to pick
 */
```

**7.2 Pattern Matching Strategy**
- Two-tier matching: keyword cluster matching (fast, pattern-based) + phrase template matching (structured, higher confidence)
- Keyword clusters: each intent/command has a set of weighted keywords. Score = sum of matched keyword weights / total possible weight
- Phrase templates: regex-like templates for common expressions (e.g., "fix {the|my|our} {tests|testing|test suite}" → `/legion:review`)
- Scoring: keyword score (0-1) * 0.6 + template match bonus (0 or 0.3) + exact command name bonus (0 or 0.1)

**7.3 Command Route Mapping**
Map NL phrases to Legion commands (not just intent flags):

| Pattern Category | Example Phrases | Routes To |
|-----------------|-----------------|-----------|
| Start/initialize | "start a new project", "initialize", "set up" | `/legion:start` |
| Plan/decompose | "plan the next phase", "break down phase 3" | `/legion:plan` |
| Build/execute | "build it", "execute the plans", "run the build" | `/legion:build` |
| Review/test | "review the code", "run tests", "check quality" | `/legion:review` |
| Status/progress | "show progress", "where are we", "what's next" | `/legion:status` |
| Quick task | "quick fix", "run a one-off task" | `/legion:quick` |
| Advise/consult | "get advice on", "ask about", "consult on" | `/legion:advise` |
| Explore/research | "explore options", "research", "crystallize" | `/legion:explore` |
| Harden | "harden security", "security audit", "fix vulnerabilities" | `/legion:build --just-harden` |
| Document | "write docs", "generate documentation" | `/legion:build --just-document` |

**7.4 Confidence Thresholds and Fallback**
- HIGH (>= 0.8): return `{ command, flags, confidence, fallbackSuggestion: null }` — caller auto-routes
- MEDIUM (0.5-0.79): return with `fallbackSuggestion` containing the matched command — caller should confirm with user
- LOW (< 0.5): return `{ command: null, flags: [], confidence, fallbackSuggestion: "top 3 suggestions" }` — caller must ask user
- When confidence is LOW, format suggestions as: "Did you mean: 1. /legion:review  2. /legion:build --just-harden  3. /legion:status"

Function to load NL patterns from `intent-teams.yaml` `nl_patterns` section (created in Task 2).
